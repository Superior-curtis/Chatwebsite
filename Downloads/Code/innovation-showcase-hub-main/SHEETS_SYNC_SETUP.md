# Google Sheets Sync - Setup Guide

## Overview
This script automatically syncs participant data from Google Forms (via Google Sheets) to Firebase Firestore.

## Prerequisites
- Google Form collecting participant submissions
- Google Sheets linked to the form responses
- Firebase project (gibc-showcase)
- Google Cloud Platform project

## Setup Steps

### 1. Google Form Setup
Create a Google Form with the following fields:
- **Devpost Username** (required)
- **Email** (optional)
- **Project Name** (optional)
- **Project URL** (optional)

Link form responses to a Google Sheet (Form > Responses > Create Spreadsheet).

### 2. Google Cloud Console Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Select or create a project
3. Enable the **Google Sheets API**:
   - Navigate to "APIs & Services" > "Library"
   - Search for "Google Sheets API"
   - Click "Enable"

### 3. Create Service Account

1. Go to "APIs & Services" > "Credentials"
2. Click "Create Credentials" > "Service Account"
3. Fill in details:
   - Name: `gibc-sheets-sync`
   - Description: `Service account for syncing Google Sheets to Firestore`
4. Click "Create and Continue"
5. Grant role: **Editor** (or custom role with Sheets read access)
6. Click "Done"

### 4. Generate Service Account Key

1. Click on the newly created service account
2. Go to "Keys" tab
3. Click "Add Key" > "Create New Key"
4. Select **JSON** format
5. Click "Create" - the key file will download

### 5. Share Google Sheet

1. Open your Google Sheet with form responses
2. Click "Share" button
3. Paste the service account email (found in the JSON key file as `client_email`)
4. Set permission to **Viewer**
5. Uncheck "Notify people"
6. Click "Share"

### 6. Configure Environment Variables

1. Copy `.env.example` to `.env`:
   ```bash
   cp .env.example .env
   ```

2. Open the downloaded JSON key file and extract:
   - `client_email` → `GOOGLE_SERVICE_ACCOUNT_EMAIL`
   - `private_key` → `GOOGLE_PRIVATE_KEY`

3. Get your Google Sheet ID from the URL:
   ```
   https://docs.google.com/spreadsheets/d/[SHEET_ID]/edit
   ```

4. Update `.env` file:
   ```env
   GOOGLE_SHEETS_ID=1A2B3C4D5E6F7G8H9I0J
   GOOGLE_SERVICE_ACCOUNT_EMAIL=gibc-sheets-sync@project-id.iam.gserviceaccount.com
   GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nMIIEvQIBADA...\n-----END PRIVATE KEY-----\n"
   FIREBASE_PROJECT_ID=gibc-showcase
   ```

### 7. Install Dependencies

```bash
npm install dotenv firebase-admin googleapis
```

Or using the provided package file:
```bash
npm install --prefix . --file package-sync.json
```

## Usage

### One-Time Sync
Run a single sync operation:
```bash
npm run sync
```

Or directly:
```bash
node syncFromSheets.js
```

### Watch Mode
Continuously sync every 5 minutes:
```bash
npm run watch
```

Custom sync interval (every 10 minutes):
```bash
npm run sync:interval
```

Or:
```bash
node syncFromSheets.js --watch --interval=10
```

## How It Works

1. **Read Google Sheets**: Fetches all rows from the "Form Responses 1" sheet
2. **Check Existing**: Looks for existing participants by Devpost username or email
3. **Create or Update**:
   - **New participant**: Creates document with unique 10-char token
   - **Existing participant**: Updates changed fields only
4. **Summary**: Displays statistics (created, updated, skipped, errors)

## Data Flow

```
Google Form → Google Sheets → This Script → Firebase Firestore
```

## Troubleshooting

### "PERMISSION_DENIED" Error
- Ensure Google Sheet is shared with service account email
- Verify Google Sheets API is enabled
- Check Sheet ID in `.env` is correct

### "Invalid credentials" Error
- Verify private key is correctly formatted with `\n` for newlines
- Ensure service account JSON is from the correct Google Cloud project

### "No data found" 
- Check sheet name is "Form Responses 1" (or update script)
- Verify range starts at row 2 (A2:D assumes headers in row 1)
- Ensure form has received responses

### Duplicate Participants
- Script checks Devpost username first, then email
- If both differ, a new participant is created
- Use Admin Dashboard to merge/delete duplicates

## Security Notes

⚠️ **Important**:
- Never commit `.env` file to version control
- Keep service account JSON key secure
- Use minimal permissions (Sheets viewer + Firestore write)
- Regularly rotate service account keys

## Advanced Configuration

### Custom Sheet Range
Edit line in `syncFromSheets.js`:
```javascript
range: "Form Responses 1!A2:E" // Add column E if you have 5 fields
```

### Add Custom Fields
Update the row parsing section:
```javascript
const customField = row[5]?.trim(); // Column F

const participantData = {
  // ... existing fields
  customField: customField || null
};
```

### Change Sync Interval
Default: 5 minutes. Adjust with `--interval` flag:
```bash
node syncFromSheets.js --watch --interval=15
```
